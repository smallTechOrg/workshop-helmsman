# Data Model

All state is single SQLite file at `./data/helmsman.db` (auto-created on boot).
Four tables. No migrations framework in Phase 1 — `Base.metadata.create_all`
runs idempotently at startup.

```sql
CREATE TABLE workshop (
  id                  INTEGER PRIMARY KEY AUTOINCREMENT,
  name                VARCHAR(200) NOT NULL,
  created_at          DATETIME NOT NULL,
  expires_at          DATETIME NOT NULL,
  admin_token         VARCHAR(64) NOT NULL UNIQUE,
  participant_slug    VARCHAR(64) NOT NULL UNIQUE,
  milestone_config    TEXT NOT NULL,        -- JSON-as-text list of {id,title,description}
  archived            BOOLEAN NOT NULL DEFAULT 0
);
CREATE INDEX ix_workshop_admin_token ON workshop(admin_token);
CREATE INDEX ix_workshop_participant_slug ON workshop(participant_slug);

CREATE TABLE participant (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  workshop_id INTEGER NOT NULL REFERENCES workshop(id) ON DELETE CASCADE,
  name        VARCHAR(120) NOT NULL,
  joined_at   DATETIME NOT NULL
);
CREATE INDEX ix_participant_workshop_id ON participant(workshop_id);

CREATE TABLE milestone_completion (
  id              INTEGER PRIMARY KEY AUTOINCREMENT,
  participant_id  INTEGER NOT NULL REFERENCES participant(id) ON DELETE CASCADE,
  milestone_id    VARCHAR(64) NOT NULL,
  milestone_title VARCHAR(200) NOT NULL,
  completed_at    DATETIME NOT NULL
);
CREATE INDEX ix_milestone_completion_participant_id ON milestone_completion(participant_id);

CREATE TABLE help_request (
  id             INTEGER PRIMARY KEY AUTOINCREMENT,
  participant_id INTEGER NOT NULL REFERENCES participant(id) ON DELETE CASCADE,
  message        TEXT NOT NULL,
  created_at     DATETIME NOT NULL
);
CREATE INDEX ix_help_request_participant_id ON help_request(participant_id);
```

## Notes

- `workshop.admin_token` and `workshop.participant_slug` are random secrets
  generated by `secrets.token_urlsafe` at creation time.
- `workshop.milestone_config` is JSON-as-text (Phase 1 keeps it simple — no
  separate milestone table; milestones mutate freely per workshop).
- `archived` is a soft-delete flag. Participant routes treat `archived=1` as
  404. Admin routes keep working until archived.
- `expires_at` only blocks participant routes. Admin dashboard always loads.
